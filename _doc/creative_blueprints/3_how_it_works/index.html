How it works

One recording in, a scored evaluation out

A single automated pipeline — no manual scoring step anywhere in the chain.

1 Upload

Submit a recording

An agent or reviewer uploads a customer-service call — audio, or video-with-audio — through the web app.

2 Store & queue

Persist & dispatch

The backend stores the media in AWS S3 and publishes an analysis job to a RabbitMQ work queue.

3 Analyze

The AI core

  • · Transcription
  • · Speaker diarization
  • · Service-quality scoring
  • · Course recommendation
4 Proxy · optional

Lightweight playback

A worker renders a 480p proxy and MP3 for fast review; the heavy original can then be purged.

5 Report

Back to the team

Results land in MongoDB and surface in the app as scored evaluations, transcripts, and recommended courses.

Backend and AI stay decoupled through the message queue — the upload returns immediately while analysis runs asynchronously in the background.
